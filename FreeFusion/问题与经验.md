## 训练过程中的报错
### 1.CUDA 索引越界错误 (index out of bounds)

    "index out of bounds" failed./pytorch/aten/src/ATen/native/cuda/ScatterGatherKernel.cu:365:       operatorblock: [530,0,0], thread: [2,0,0] Assertion idx dim >= 0 && idx_dim < index_size && "index out of bounds" failed

计算损失时添加补丁

    # A. 拿到网络预测的合法类别上限 (比如 9 分类，合法的最大数字就是 8)
            max_valid_idx = opt.TRAINING.NUM_CLASSES - 1
            
            # B. 找出那些“既不是 255 (因为你的交叉熵设置了忽略255)，又大于合法上限”的异常脏数据
            invalid_mask = (target_seg > max_valid_idx) & (target_seg != 255)
            
            # C. 强制把这些惹祸的脏数据截断到最大合法数字，同时清理掉可能存在的负数
            target_seg[invalid_mask] = max_valid_idx
            target_seg[target_seg < 0] = 0
            
            # D. 针对作者自己写的 DiceLoss 的终极保命策略：
            # 很多开源的 DiceLoss 遇到 255 会在底层的 one_hot 转换时崩溃，所以我们喂给 Dice 的标签单独处理一下
            safe_target_seg = target_seg.clone()
            safe_target_seg[safe_target_seg == 255] = 0 # 临时把 255 换成合法的 0，骗过底层的张量转换机制

### 3.potsdam和whu数据集图片分辨率过大

potsdam数据集原始图片分辨率为6000*6000，而whu达到了32507 × 15345像素。GPU显存不够，需额外裁剪

### 4.计算SSIM时浮点数范围未知

    ValueError: Since image dtype is floating point, you must specify the data_range parameter...

降低scikit版本，旧版本可自动猜浮点数范围，但是又产生了报错，原因是2.x版本的numpy与很多老库不兼容，于是

    pip install --upgrade "numpy<2.0"

之后即可解决
